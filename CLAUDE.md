### 运营约束
- **严格匿名**：绝不暴露真实身份（不暴露皮下），不在线下露面
- **小体量运营**：不具备与商家谈合作的体量，不搞线下物料（贴海报、发传单等不现实）
- **不走官方渠道**：无法通过辅导员、班长、企业微信等学校官方途径推广
- **内容驱动增长**：无法依赖活动裂变或他人主动转发，主要靠自己主动挖掘内容

### 目标用户特点（上海立信会计金融学院）
- **大三大四学生几乎不刷微信朋友圈**，朋友圈征集对他们无效
- **微信为主，QQ 基本不用**（仅新生用 QQ 较多），QQ 校园墙不适用
- 不会主动参与投稿类活动，不会主动转发活动给大四学生
- 对毕业照有展示意愿，但需要降低参与门槛

### 当前可行的运营策略
- **手动私聊要授权**：在微信朋友圈刷到毕业照 → 私聊问能否使用（话术用「小狐」身份，三步走：自我介绍 → 说明来意 → 给台阶下）
- **小红书评论区征集**：发笔记号召评论区带图投稿（小红书已支持评论区发图片，需实名+注册满7天）
- **合集类内容**：主动收集照片做成合集发布（朋友圈「小狐收集」系列 + 小红书「立信毕业照合集」笔记）
- **主动挖掘 > 被动等投稿**：自己去刷、去问、去收集，而不是等用户主动来
- **一次收集，双平台发布**：微信朋友圈 + 小红书同步使用
- **小红书评论区暖场**：发征集笔记后用小号先评论带图，打破空场，带动从众效应

### 微信私域行业现状（2025-2026）
- **市场趋于饱和**：大多数高校已有多个"墙"类账号，新入局竞争激烈，业界出现"2026 不要再做表白墙"的声音
- **学校监管收紧**：越来越多高校出台管理规定，封禁非官方账号，合规化运营是趋势
- **从个人微信转向企业微信**：企微支持实名认证、群管理权限、消息互通，成为校园私域标配
- **小程序 + 私域双循环**：圈子+话题+帖子+商城四维一体，构建"内容-社交-交易"校园小生态
- **AI 赋能**：AI 自动回复、内容生成、情感分析预警等深度嵌入校园私域运营
- **分站/合伙人模式**：选拔学生 KOC 作为分站站长，收益共享，跨校裂变扩张
- **核心公式**：校园墙价值 = 覆盖该校学生比例 × 信任度 × 变现能力

### 小红书自动化工具
- 所有脚本位于 `xhs_tools/` 目录下
- **`xhs_tools/xhs_login.py`**：手动扫码登录，保存 auth 到 `secrets/xhs_auth.json`，同时保存用户信息到 `secrets/xhs_profile.json`
  - `python xhs_tools/xhs_login.py` — 登录
  - `python xhs_tools/xhs_login.py --check` — 检查登录是否过期
- **`xhs_tools/xhs_monitor.py`**：打开已登录的可见浏览器，手动查看评论/通知
  - `python xhs_tools/xhs_monitor.py` — 打开个人主页
  - `python xhs_tools/xhs_monitor.py noti` — 打开通知页
- **`xhs_tools/xhs_analyze.py`**：自动抓取所有笔记数据（点赞数、标题等），生成分析报告，保存到 `运营数据/`
- **`xhs_tools/xhs_competitors.py`**：批量抓取竞品账号数据，对比分析
- **注意**：小红书 API 有严格反爬，直接 HTTP 请求会被封。数据抓取只能通过 Playwright 浏览器 + 拦截 API 响应实现。Cookie 过期后需重新扫码

## 可用 MCP 工具

### Playwright（浏览器自动化）
- 可以打开网页、点击、填表单、截图等，模拟真实浏览器操作
- **适用场景**：需要登录后才能访问的页面、JavaScript 渲染的动态页面、网页交互操作
- **注意**：操作需要时间，复杂任务请给足超时时间

### Firecrawl（网页抓取）
- 搜索、抓取网页内容，返回 Markdown 格式的结构化数据，支持批量处理和绕过反爬
- **适用场景**：搜索信息、批量抓取网页、获取 LLM 友好的内容
- **免费额度**：500 credits（一次性）
